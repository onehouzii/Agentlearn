# 学习计划（第一性原理版）

## 核心原理
不是"先学完所有技术再做项目"，而是"从第一天就用AI做项目，在项目中学习"。

这个岗位的本质是：**用AI工具完成软件开发交付**。

---

## 底层能力模型（第一性原理拆解）

| 底层能力 | 具体表现 | 为什么重要 |
|---------|---------|-----------|
| **AI编程思维** | 会用Cursor/Claude，会写Prompt | 岗位核心工作方式 |
| **代码理解能力** | 能读懂AI生成的代码，能调试 | AI不会100%正确 |
| **需求拆解能力** | 把业务需求拆成可执行任务 | ToB项目核心能力 |
| **技术栈基础** | Django/Vue/API基本概念 | 能与AI有效协作 |
| **项目交付能力** | 从0到1完整交付项目 | 证明你能干活 |

---

## 学习路线（项目驱动）

### 第1周：建立AI编程工作流（最重要）

**目标**：学会用AI写代码，而不是自己从零写

**每日任务**：
- [ ] Day 1: 安装Cursor IDE，用Claude API
- [ ] Day 2: 用AI重构你以前的微信小程序代码（练手）
- [ ] Day 3: 学习Prompt技巧："分步思考"、"代码审查"、"错误修复"
- [ ] Day 4-5: 用AI写一个简单的Python脚本（如爬虫、数据处理）
- [ ] Day 6-7: 总结你的Prompt模板库

**验证标准**：你能明显感觉到"AI帮你写80%，你只需要思考和调整20%"

---

### 第2-3周：项目一 - 任务管理系统（Django + Vue）

**目标**：用AI完成一个全栈项目，同时学习Django和Vue

**不是先学再做，而是边做边学**：

**阶段1：后端Django（用AI驱动）**
- [ ] 用AI生成Django项目骨架
- [ ] 用AI设计数据模型（Task, User）
- [ ] 用AI写CRUD API
- [ ] 用AI写用户认证
- [ ] **关键**：每一步都要求AI解释代码，你要读懂

**阶段2：前端Vue（用AI驱动）**
- [ ] 用AI生成Vue项目骨架
- [ ] 用AI写任务列表页面
- [ ] 用AI写任务添加/编辑页面
- [ ] 用AI接入后端API
- [ ] 用AI加ECharts图表（满足优先条件）

**阶段3：项目整合**
- [ ] 前后端联调
- [ ] 用AI修复bug
- [ ] 写README文档

**交付物**：一个完整可运行的任务管理系统

---

### 第4周：项目二 - AI知识库助手（RAG/Agent）

**目标**：满足岗位的"AI Agent/RAG落地经验"优先条件

**做法**（还是用AI驱动）：
- [ ] 用AI搜索RAG/Agent的基本概念
- [ ] 用AI找一个简单的开源RAG项目
- [ ] 用AI帮你跑通这个项目
- [ ] 用AI帮你改造成"金融投研助手"或"留学教育知识库"（匹配岗位方向）
- [ ] 用AI加一个简单的前端界面

**关键**：不要纠结底层算法细节，重点是"能用AI把东西做出来"

---

### 第5周：作品集准备 + 面试模拟

**作品集整理**：
- [ ] 把所有项目传到GitHub
- [ ] 每个项目写清晰的README（功能、技术栈、如何运行）
- [ ] 录一个简短的演示视频（每个项目2-3分钟）
- [ ] 部署到云服务器（可选，用AI帮你部署）

**面试准备**：
- [ ] 用AI模拟面试："你是面试官，根据这个JD和我的简历，问我技术问题"
- [ ] 准备3个"我用AI解决了什么问题"的故事
- [ ] 准备"为什么想做AI编程"的回答

---

## 每日工作模板（AI编程模式）

每天开始时：
```
1. 明确今天要完成什么功能
2. 用AI生成代码
3. 读懂AI生成的代码
4. 运行、调试、修改
5. 让AI解释你不懂的地方
6. 记录今天学到的知识点
```

---

## 关键认知（第一性原理）

1. **你不是AI的替代品，而是AI的指挥者**
   - 重点思考"做什么"，而不是"怎么写"

2. **不需要背语法，只需要理解概念**
   - AI记得所有语法，你只需要知道"有这个东西"

3. **项目比知识点重要**
   - 有3个完整项目 > 背完10本技术书

4. **最快的学习方式：用AI教你**
   - 遇到问题先问AI："解释一下这个概念，用我能懂的话"
